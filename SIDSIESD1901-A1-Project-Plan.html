Overview

MathWars is a free to use website designed to teach kids and teens that Math can be fun and engaging. Students will be able to practice solving the many problems in Mathematics ranging from easy to mind-boggling. Each successful solution will be rewarded with points; the harder the question, the more points will be awarded when solved. This online platform aims to be community driven, which means that anyone can set up their own questions with answers that others can try to solve and gain points from. There are groups that the users can join to solve large problems together to each earn more points than usual. The skills taught in MathWars will be very useful in many different fields that involve complex problem solving.

Game Data

MathWars will have static game data containing information related to the user’s name, age, date of birth, home address, and ID number:

  • Name: the datatype of name is String because names are “Tom Jerry” or “Somethea which fits perfectly into a string.
  • Age & ID No.: the datatype for these two is a Number because ages are 23 or 54 a increase by one every year. Number is good for age and ID because we can perform Mathematical operations on it like plus and minus. Ages can be increased by one every year and unique IDs can be generated from increasing the highest ID number by one.
  • Date Of Birth & Address: I chose the datatype for the date of birth to be an Object rather than a string because with strings, the format might not be consistent unlike objects which will always have the keys “day”, “month”, and “year” followed by a semi-colon “:” and its value in string.

This is another JSON format that keeps a record of phone numbers that includes the type and number:

  • Phone: an Array that contains a list, which is the right data type ot store a list of phone numbers.
  • Type & Number: are both inside an Object acting as the key that contains the string values.

Global Variables

The dynamic information needed throughout MathWars is the player’s score, ranking, current exercise/lesson page, cash, friend list, and accuracy. As the user continues to use, these values will change all the time:

  • Score: A Number because it is easier to increase or decrease the score simply by adding or subtracting values.
  • Ranking: A Number because it is easier to increase or decrease the user’s rank when they win or lose simply by adding or subtracting numbers.
  • Current Exercise/Lesson Page: An Object because the main keys of this object will be things like name, description, number, and so on. The values of the keys name and description will be strings whereas the page number will be a number type.
  • Cash: A Number because it is easier to increase or decrease the cash when the user uses or earns coins by adding or subtracting values when the transactions take place
  • Friend List: An Array because iit is easier to contain the list of friends that can be expanded or made smaller easily by using “push” or “splice”.
  • Total Win: A Number because it is easier to increase the number by one every time a victory is achieved.
